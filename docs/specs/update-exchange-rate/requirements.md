# Requirements Document — 為替レート更新（内部 API）

## Introduction

Tateca は旅行・食事・イベントなど日常の割り勘シーンにおける共有費用を管理するアプリケーションである。
為替レート更新は、外部為替レートサービスから最新のレートを取得し、データベースに保存する内部操作である。

この操作はスケジューラ（EventBridge + Lambda）によって毎日 UTC 00:01 に自動実行される。
取得したレートは当日分と翌日分の 2 日分として保存され、翌日分は事前作成として機能する。

## Scope

- 外部サービスからの最新為替レート取得
- 当日分と翌日分の為替レート保存（新規作成または更新）
- マスタデータに存在しない通貨のスキップ
- レートが変更されていない場合の更新スキップ（冪等性）

### Out of Scope

- 特定日付を指定した為替レート取得（常に最新レートを取得）
- 為替レートの削除
- 外部サービスのリトライ制御（API クライアント層の責務）
- スケジューリング制御（EventBridge の責務）
- 外部サービスが返却する為替レート値の妥当性検証（0 や負の値の検証は行わない）

## Common Preconditions

すべての要件は以下を暗黙の前提とする。

- 要求者は API キーで認証済みである（内部システムからのアクセス）

## Domain Definitions

- **為替レート:** ある通貨の日本円に対する交換比率。通貨コードと日付の組み合わせで一意に特定される
- **通貨マスタ:** システムに登録されている通貨の一覧。外部サービスから取得したレートのうち、マスタに存在する通貨のみが保存対象となる
- **外部為替レートサービス:** 最新の為替レートを提供する外部 API
- **事前作成:** 翌日分のレートを前日に作成しておくこと。翌日の更新時に上書きされる

## Processing Order

1. 外部サービスから最新の為替レートを取得する
2. 当日日付で為替レートを保存する（既存の場合はレート変更時のみ更新）
3. 翌日日付で為替レートを保存する（既存の場合はレート変更時のみ更新）
4. 新規作成分を一括保存する（既存レートの更新は変更検知により自動反映）

## Requirements

### Requirement 1: 為替レートの新規保存

**User Story:** システム管理者として、最新の為替レートを自動的に取得・保存したい。ユーザーが最新のレートで取引を記録できるようにするため。

#### Acceptance Criteria

1. GIVEN マスタデータに通貨が登録されており、当日および翌日の為替レートが存在しない WHEN 為替レート更新を実行する THEN システム SHALL 外部サービスから取得したレートを当日分と翌日分として保存する

2. GIVEN 外部サービスが 3 通貨のレートを返却し、すべてがマスタに存在する WHEN 為替レート更新を実行する THEN システム SHALL 6 件（3 通貨 × 2 日分）のレコードを保存する

### Requirement 2: 既存レートの更新

**User Story:** システム管理者として、為替レートの変動をデータベースに反映したい。

#### Acceptance Criteria

1. GIVEN 当日および翌日の為替レートが既に存在し、外部サービスが異なるレートを返却する WHEN 為替レート更新を実行する THEN システム SHALL 既存のレートを新しい値で更新する

2. GIVEN 当日および翌日の為替レートが既に存在し、外部サービスが同じレートを返却する WHEN 為替レート更新を実行する THEN システム SHALL 更新をスキップする（更新日時も変更しない）

### Requirement 3: 未知通貨のスキップ

**User Story:** システム管理者として、マスタに登録されていない通貨のレートを無視したい。データの整合性を保つため。

#### Acceptance Criteria

1. GIVEN 外部サービスがマスタに存在しない通貨のレートを含む WHEN 為替レート更新を実行する THEN システム SHALL 未知の通貨をスキップし、マスタに存在する通貨のレートのみを保存する

### Requirement 4: 外部サービス障害

**User Story:** システム管理者として、外部サービス障害時にデータの整合性を保ちたい。

#### Acceptance Criteria

1. GIVEN 外部為替レートサービスが利用不可である WHEN 為替レート更新を実行する THEN システム SHALL エラーとして処理を中断し、データベースへの変更をロールバックする

### Requirement 5: 空レスポンスの処理

**User Story:** システム管理者として、外部サービスが空のレスポンスを返した場合でもシステムが正常に動作することを確認したい。

#### Acceptance Criteria

1. GIVEN 外部サービスが空の為替レート一覧を返却する WHEN 為替レート更新を実行する THEN システム SHALL エラーを発生させず、保存件数として 0 を返却する

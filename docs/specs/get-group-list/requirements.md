# Requirements Document — グループ一覧取得

## Introduction

Tateca は旅行・食事・イベントなど日常の割り勘シーンにおける共有費用を管理するアプリケーションである。
グループ一覧取得は、認証済みユーザーが自分が所属するグループの一覧を取得する操作である。

## Scope

- 認証済みユーザーが所属するグループの一覧取得

### Out of Scope

- グループ一覧のページネーション
- グループ一覧のソート・フィルタリング
- グループ一覧のキャッシュ

## Common Preconditions

すべての要件は以下を暗黙の前提とする。

- 要求者は認証済みである

## Domain Definitions

- **所属グループ:** 認証済みユーザーが認証済みメンバーとして参加しているグループ。脱退済み（認証紐付け解除済み）のグループは含まれない

## Requirements

### Requirement 1: グループ一覧の取得（正常系）

**User Story:** 認証済みユーザーとして、自分が所属するグループの一覧を確認したい。管理するグループを把握するため。

#### Acceptance Criteria

1. GIVEN 要求者が複数のグループに所属している WHEN グループ一覧の取得を要求する THEN システム SHALL 所属する全グループの情報を返す

2. GIVEN 要求者がどのグループにも所属していない WHEN グループ一覧の取得を要求する THEN システム SHALL 空のリストを返す

3. GIVEN 要求者がグループを脱退済みである WHEN グループ一覧の取得を要求する THEN システム SHALL 脱退済みグループを一覧に含めない

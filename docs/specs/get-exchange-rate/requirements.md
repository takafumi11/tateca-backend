# Requirements Document — 為替レート取得

## Introduction

Tateca は旅行・食事・イベントなど日常の割り勘シーンにおける共有費用を管理するアプリケーションである。
為替レート取得は、指定した日付の為替レート一覧を取得する操作である。

ユーザーは取引の登録や確認時に、各通貨の日本円に対する為替レートを参照する。
有効な通貨のレートのみが返却され、無効化された通貨のレートは除外される。

## Scope

- 指定日付の有効通貨の為替レート一覧取得
- 為替レートが存在しない日付の場合は空の一覧を返却

### Out of Scope

- 為替レートの作成・更新（内部 API の責務）
- 特定通貨のみを指定した取得（常に全有効通貨を返却）
- 為替レートのキャッシュ
- 過去・未来の日付に対するアクセス制限（任意の日付を指定可能）
- 返却一覧の並び順の保証

## Common Preconditions

すべての要件は以下を暗黙の前提とする。

- 要求者は認証済みである

## Domain Definitions

- **為替レート:** ある通貨の日本円に対する交換比率。日付と通貨コードの組み合わせで一意に特定される
- **有効通貨:** システムで利用可能として登録されている通貨。無効化された通貨は為替レート取得から除外される
- **通貨情報:** 通貨コード・日本語通貨名・英語通貨名・日本語国名・英語国名・通貨記号・記号位置を含む通貨のメタデータ

## Requirements

### Requirement 1: 為替レート一覧の取得

**User Story:** 認証済みユーザーとして、特定の日付の為替レートを確認したい。取引に適切な為替レートを適用するため。

#### Acceptance Criteria

1. GIVEN 指定された日付に有効通貨の為替レートが存在する WHEN 為替レート一覧の取得を要求する THEN システム SHALL 各通貨の為替レートと通貨情報を含む一覧を返却する

2. GIVEN 指定された日付に為替レートが存在しない WHEN 為替レート一覧の取得を要求する THEN システム SHALL 空の一覧を返却する

3. GIVEN 指定された日付において有効通貨と無効通貨の両方に為替レートが存在する WHEN 為替レート一覧の取得を要求する THEN システム SHALL 有効通貨のレートのみを返却する

### Requirement 2: 入力バリデーション

**User Story:** 認証済みユーザーとして、不正な入力に対して適切なエラーを受け取りたい。問題の原因を理解するため。

#### Acceptance Criteria

1. GIVEN 認証済みユーザー WHEN 不正な形式の日付で為替レート一覧の取得を要求する THEN システム SHALL 入力不備として拒否する

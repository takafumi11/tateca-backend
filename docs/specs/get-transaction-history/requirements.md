# Requirements Document — 取引履歴取得

## Introduction

Tateca は旅行・食事・イベントなど日常の割り勘シーンにおける共有費用を管理するアプリケーションである。
取引履歴取得は、グループ内の取引一覧を新しい順に取得する操作である。

取得件数は指定可能であり、未指定の場合は既定の件数（5件）が返却される。
各取引の概要情報（取引種別・タイトル・金額・通貨情報・日付）が含まれるが、義務リストや受取人の詳細は含まれない。

## Scope

- グループ内の取引一覧の取得
- 取得件数の指定（既定値: 5件）
- 作成日時の降順（新しい順）でのソート

### Out of Scope

- グループメンバーシップの検証（現在の実装では、認証済みであれば任意のグループの取引履歴を取得可能）
- グループの存在確認（存在しないグループを指定した場合も、取引が0件のグループと同様に空の一覧を返却する）
- 取得件数の範囲バリデーション（0以下や極端に大きい値の検証は行わない）
- ページネーション（オフセットベース・カーソルベース）
- 取引種別によるフィルタリング
- 日付範囲によるフィルタリング
- 各取引の義務リストや受取人の詳細（概要情報のみ返却される。詳細は取引詳細取得で対応）

## Common Preconditions

すべての要件は以下を暗黙の前提とする。

- 要求者は認証済みである

## Domain Definitions

- **取引履歴:** グループ内の取引を作成日時の降順に並べた一覧
- **取引概要:** 取引の識別子・取引種別・タイトル・金額・通貨情報・日付を含む情報。義務リストや受取人の詳細は含まない
- **既定取得件数:** 取得件数が指定されなかった場合に適用される件数。5件

## Requirements

### Requirement 1: 取引履歴の取得

**User Story:** 認証済みユーザーとして、グループの最近の取引を確認したい。費用の状況を把握するため。

#### Acceptance Criteria

1. GIVEN 指定されたグループに取引が存在する WHEN 取引履歴の取得を要求する THEN システム SHALL 作成日時の降順で取引概要の一覧を返却する

2. GIVEN 指定されたグループに取引が存在する WHEN 取得件数を指定して取引履歴の取得を要求する THEN システム SHALL 指定された件数を上限として取引概要の一覧を返却する

3. GIVEN 指定されたグループに取引が存在する WHEN 取得件数を指定せずに取引履歴の取得を要求する THEN システム SHALL 既定取得件数（5件）を上限として取引概要の一覧を返却する

4. GIVEN 指定されたグループに取引が存在しない WHEN 取引履歴の取得を要求する THEN システム SHALL 空の一覧を返却する

5. GIVEN 取引履歴の取得が完了した WHEN 各取引概要を確認する THEN システム SHALL 取引識別子・取引種別・タイトル・金額・通貨情報・日付が含まれていること

### Requirement 2: 入力バリデーション

**User Story:** システム管理者として、不正な入力による取引履歴取得を防ぎたい。

#### Acceptance Criteria

1. WHEN グループ識別子が不正な形式である THEN システム SHALL 入力不備として拒否する
